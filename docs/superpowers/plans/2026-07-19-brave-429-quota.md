# Brave 429 Quota Exhaustion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist Brave's proven long-window HTTP 429 exhaustion state instead of retaining stale remaining quota.

**Architecture:** Extend the Brave HTTP parser with a strict aligned-bucket classifier used only for 429 responses. A uniquely identified exhausted longest window returns a normal `QuotaResult`; short-window and malformed 429 responses continue to throw `QuotaError.rateLimited`, preserving existing transient-failure behavior.

**Tech Stack:** Swift 5, Foundation, existing shell behavior harness, SwiftPM macOS build.

---

- [ ] **Step 3: Implement strict policy parsing and 429 classification**

In `QuotaService.swift`:

```swift
private struct BravePolicyBucket {
    let quota: Int
    let window: Int
}
```

Add a strict parser that accepts only comma-separated `<positive quota>;w=<positive duration>` items. In `parseBraveHTTPResponse`, add `now: Date = Date()` and for HTTP 429:

1. Parse limit, remaining, and strict policy arrays.
2. Require equal non-zero counts, non-negative limit/remaining values, each policy quota equal to its aligned limit, and one unique greatest policy duration.
3. If the longest remaining is zero and its limit is positive, return `QuotaResult(remaining: 0, limit: ..., resetAt: alignedReset, quotaLabel: "0 / ... monthly requests", quotaText: .localized(.monthlyRequestsFormat, "0", "..."), httpStatus: 429, diagnosticMessage: "Brave long-window request quota exhausted.", diagnosticText: .localized(.braveQuotaExhaustedDiagnostic))`.
4. If the longest remaining is positive, throw `QuotaError.rateLimited` using the earliest non-negative aligned reset among zero-remaining shorter windows.
5. On invalid evidence, throw `QuotaError.rateLimited(resetAt: nil)`.

Reset arrays do not participate in proving exhaustion. They produce a date only when their count aligns and the selected value is non-negative.

In `AppLanguage.swift`, add `braveQuotaExhaustedDiagnostic` to `L10n.Key`, persisted-status restoration, legacy raw diagnostic restoration, and all five language tables.
